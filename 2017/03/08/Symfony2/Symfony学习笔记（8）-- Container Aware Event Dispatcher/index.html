Symfony学习笔记（8）-- GenericEvent

Mar 8, 2017

在调用dispatcher的dispatch方法时如果不给其传入一个自定义的Event对象，那么Dispatcher会自动创建一个默认的Event对象。而GenericEvent的引入是为了在整个程序中都只使用一个事件对象的情况。GenericEvent封装了一个事件主题“subject”。并且其还拥有如下简洁的API:

  • __construct() 构造器可以接收事件主题和任何参数

  • getSubject() 获取主题 setArgument() 通过键设置一个参数

  • setArguments() 设置一个参数数组

  • getArgument() 通过键获取一个参数值

  • getArguments() 获取所有参数值 hasArgument() 如果某个键值存在，则返回true。

GenericEvent同时还在参数集上实现了ArrayAccess，所以可以非常方便的通过传入额外的参数。
