Cycle.js 回顾

Cycle.js 是一个函数响应式（FRP）前端框架，出来也算有些年头了，但函数式编程、响应式编程抬升了一些 Cycle.js 的入门门槛，所以即便 stalz 这几年都在各地做推广，其提出的理念和模型也获得了许多大牛开发者的认可，该框架的热度也只是尚可。

Cycle.js 定义了一个环形的应用模型，这也是 Cycle.js 的由来：

环形模型只有两个元素：

  • 外部世界（External World），这里充斥着副作用（Side Effect），比如 HTTP，DOM，hash history，localstorage 等等。
  • 逻辑部分（Logic），在 Cycle.js 中，逻辑部分被装入一个 main 函数中，开发者应当尽可能让 main 成为一个纯函数（pure function）。

因此，环形模型中，也就只存在 逻辑 和 外部世界 的交互。对于逻辑部分来说，它的输入输出分别为：

  • 输入：逻辑需要接收外部世界信息，因此，逻辑的输入被称为 读副作用（Read Effects）。在 Cycle.js 中，习惯以 sources 命名读副作用。
  • 输出：逻辑需要向外部世界反馈信息，因此，逻辑的输出被称为 写副作用（Write Effects）。在 Cycle.js 中，习惯以 sinks 命名写副作用。
